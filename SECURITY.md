# Security policy

Security fixes are considered for the latest Tabento — Locus Legacy Edition release. Historical tags are retained but are not separately maintained.

Do not open a public issue for a vulnerability that could expose user data, enable script injection, weaken import validation, or abuse extension permissions. Use GitHub’s **Report a vulnerability** private advisory form. If private advisories are unavailable, contact the repository owner through the email published on the GitHub profile with “Tabento security” in the subject.

Include the affected version, reproduction steps, impact, and any suggested fix. Never attach real user exports or private browsing data. Relevant reports include stored/DOM injection, unsafe restore behavior, unintended disclosure, permission escalation, insecure navigation, and release-packaging problems.
